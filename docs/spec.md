## 1. サイト概要

### 目的・解決する課題

株式会社CIMAはイベント業界（ウェディング関連）の企業。個人・法人の両方をターゲットとし、以下を目的としたコーポレートサイトを制作する。

- **問い合わせ獲得**: フローティングCTAと複数導線でお問い合わせフォームへ誘導する
- **ブランディング**: イベント業界にふさわしい上質かつ親しみやすいデザインで信頼感を構築する
- **信頼構築**: 実績・お客様の声・代表あいさつで安心感を提供する

### 対象ユーザー

| ターゲット | 説明 |
|-----------|------|
| 個人（personal） | ウェディングを検討している個人のお客様 |
| 法人（corporate） | イベント企画を外部委託したい企業 |

### 技術スタック

| 項目 | 技術 |
|------|------|
| マークアップ | HTML5 + CSS3（静的サイト） |
| スタイル | CSSカスタムプロパティ（CSS変数）ベース |
| フォント | Google Fonts: Noto Serif JP（本文）、Noto Sans JP（UI補助） |
| ホスティング | Vercel（静的デプロイ） |
| フォーム送信 | 外部サービス連携（Formspree / Netlify Forms / Google Forms など要確認） |
| SNS連携 | Instagram埋め込み |

## 2. 完成形の定義

### 完成の条件

- 全12ページ＋1カスタムページが正常に表示される
- PC（1280px以上）・タブレット（768px〜1279px）・モバイル（767px以下）の3ブレークポイントでレイアウト崩れがない
- フローティングCTAが全ページで常時表示される
- お問い合わせフォームが正常に送信できる
- ハンバーガーメニューがタブレット・モバイルで正常に開閉する
- 全セクションのアニメーション（フェードイン・ホバースケール）が動作する
- ヒーローセクションのパララックスが動作する
- 動画セクションにYouTube埋め込みまたはプレースホルダーが表示される
- Google Mapが埋め込まれている
- Instagram連携が表示されている
- OGP / meta情報が全ページに設定されている

### MVPの定義

- 全ページのHTML/CSSが完成し、プレースホルダー画像・テキストの状態で公開可能
- レスポンシブ対応済み
- お問い合わせフォームが動作する
- ハンバーガーメニューが動作する

### スコープ外

- CMS（WordPress等）への移行
- 会員機能
- ECカート・決済機能
- 多言語対応
- メールマガジン配信システム

### カラーパレット

|--------|-----|------|
| `--main` | `#c13e3e` | メインカラー。見出し・ボタン・ラベル |

### フォント

| フォント | ウェイト | 用途 |
|---------|---------|------|
| Noto Serif JP（明朝体） | 400, 700, 900 | 本文・見出し・キャッチコピー。`fontFamily: "mincho"` 指定 |
| Noto Sans JP（ゴシック体） | 300, 400, 500, 700, 900 | フォーム・ボタン・ラベルなどUI要素 |

### デザインテイスト

- **friendly**（親しみやすい）: 角丸を多用し、暖色系カラー（赤・オレンジ）で温かみを出す
- 明朝体をベースフォントにすることで上品さとイベント業界らしい格式を両立する

### 角丸

| 用途 | 値 |
|------|-----|
| カード・画像フレーム | `12px`（`--radius`） |
| ボタン（ヘッダー・ヒーロー） | `12px` |
| フォームフィールド | `8px` |
| バッジ・タグ | `12px` |

## 10. アニメーション仕様

| 項目 | 値 |
|------|-----|
| ホバーエフェクト | `scale`。カード・ボタンホバー時に`transform: scale(1.03)` |
| ページロード | `fade`。各セクションがビューポートに入った時にフェードイン |
| アニメーション速度 | `0.6s`（`transition-duration: 0.6s`） |
| 強度 | `normal` |
| パララックス（グローバル） | 無効。ただしヒーロー・サービスセクションは個別に有効 |
| タイピングアニメーション | なし |
| カウンターアニメーション | なし |
| プログレスバー | なし |

### IntersectionObserver によるフェードイン

```
- 全セクションに `.fade-in` クラスを付与
- ビューポートの20%に入った時点で `.is-visible` を追加
- opacity: 0 → 1, translateY: 20px → 0 のトランジション
- duration: 0.6s, easing: ease-out
```

### コンポーネント依存関係

```
全ページ
├── css/variables.css ← 全スタイルが依存。変数変更で全体に波及
├── css/style.css ← variables.css に依存
├── css/components.css ← variables.css に依存
├── css/animations.css ← variables.css に依存
├── css/responsive.css ← style.css, components.css に依存
├── js/hamburger.js ← ヘッダーHTMLの構造に依存
└── js/scroll-animation.js ← .fade-in クラスの存在に依存

index.html
├── js/parallax.js ← ヒーロー・サービスセクションの背景画像要素に依存
└── 全セクション ← 各セクションのHTML構造が個別ページと共通であること

contact.html
└── js/form-tabs.js ← inquiry/document タブのHTML構造に依存
```

### 変更時の連動チェックリスト

| 変更対象 | 確認が必要なファイル |
|---------|-------------------|
| `css/variables.css` のカラー変数 | **全HTMLファイル**。インラインstyleでハードコードされている色がある場合は同時修正 |
| ヘッダーHTMLの構造 | **全HTMLファイル**（全ページに同じヘッダーがコピーされている） |
| フッターHTMLの構造 | **全HTMLファイル**（全ページに同じフッターがコピーされている） |
| ナビゲーションのリンク追加 | **全HTMLファイルのヘッダーnav** + **フッターリンク** |
| `js/hamburger.js` | ヘッダーHTML内のクラス名・ID |
| `.fade-in` / `.is-visible` | `css/animations.css` + `js/scroll-animation.js` |
| フォームフィールドの追加・削除 | `contact.html` + `js/form-tabs.js` + フォーム送信先の設定 |

### 既存HTMLのインラインスタイル問題

現在の既存HTMLは**大量のインラインstyle属性**を使用している。これはCSS変数との連動を困難にするため、実装時に以下の方針で対応する:

1. インラインstyleの色指定（`color:#1f2937`, `background:#c13e3e` 等）をCSS変数に置き換える
2. インラインstyleのレイアウト指定（`display:grid`, `padding` 等）をCSSクラスに移行する
3. インラインstyleの完全除去は段階的に行い、一度に全ページを壊さない
